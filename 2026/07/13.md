### [Show HN: Mindwalk – Replay coding-agent sessions on a 3D map of your codebase](https://news.ycombinator.com/item?id=48878682)
Mindwalkはコーディングエージェントのセッションを3Dマップ上で再生する可視化ツールです。
リポジトリを夜間マップとして描画し、エージェントが検索・読み取り・編集した箇所が光ります。
Claude CodeやCodexセッションログを読み込むスタンドアロンのGoバイナリで、セッションデータは端末上にのみ保存されます。
エージェントの作業を視覚的に理解することができ、プルリクエストレビューのように検証が可能になります。
エージェントの監査ツールはAIが実際の仕事を担当する際の重要な要素として認識されています。

### [Claude Code Artifactsを試したら、Team/Enterprise限定で自分には使えなかった話](https://zenn.dev/lnest_knowledge/articles/claude-code-artifacts-verification)
Claude Code Artifactsはコーディングセッションをレビュー用に分離し、チームメンバーが共有スペースで検証できる機能です。
Team/Enterprise プランに限定されており、個人ユーザーは利用できません。
バージョン履歴によりレビュアーは変更を比較し、成果物がレディかどうかを判断できます。
外部データベースと連携していない初期段階のLive Artifactsと異なり、Claude Code artifactsはセッション状態を反映しています。
ただし見た目は完成していても、基盤となる作業が未検証の可能性があるため、セッション記録と仮定の検討が重要です。

### [M365 Copilotの3段脆弱性「SearchLeak」を監査ログで検知する](https://qiita.com/kai_kou/items/e0ad04a969c099086f51)
Microsoft 365 CopilotのSearchLeak脆弱性（CVE-2026-42824）はパラメータ・プロンプトインジェクションと古典的なウェブセキュリティバグの組み合わせです。
攻撃者は１クリックでメール、MFAコード、SharePoint文書などの組織データを盗聴できます。
脆弱性チェーンは、URLクエリパラメータからプロンプトインジェクション、HTML レンダリングレース条件、Bing経由のCSPバイパスの3段階です。
Enterprise Copilotユーザーの監視には、エンコードされたペイロードを含むURLやBing画像エンドポイントへの異常なアウトバウンド通信を監視することが推奨されます。
Microsoftは修正前に悪用されなかったと報告していますが、企業は適切な監査ログ設定が必要です。
